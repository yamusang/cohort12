## 오늘의 핵심
- Parallelization[병렬화] : 서로 독립적인 작업 또는 나눌 수 있는 작업을 동시에 실행하여 전체 처리 시간을 단축하는 기법

## 회고 모음
- {{임정수}}: AI는 닌자를 좋아하는 것 같습니다.
- {{양승준}}: 랭그래프로 병렬처리하는법을 배웠다. 
- {{임은상}}: 서로 병렬인 작업을 동시에 돌릴 수 있었다.
- {{박성빈}}: 어느 분야든 멀티태스킹이 되는건 많은 차이를 만들어낸다.
- {{곽성빈}}: LangGraph의 Fan-out/Fan-in 패턴을 활용한 병렬 워크플로우.
- {{이영근}}: 각 llm(node)들을 병렬로 독립실핼 시켜서 수집자가 결과를 수집하여 리턴.
- {{김동국}}: 서로 독립적인 작업을 동시에 진행해서 처리 시간을 단축 시킨다는 것.
- {{윤지연}}: 내 머리로는 도저히 LLM의 속도를 따라잡을 수 없다. 그러나 그 속도를 잘 이용하는 사람이 되고 싶다.
- {{김민서}}: Gemini Flash + LangGraph 병렬 노드로 다중 응답을 생성·통합하는 파이프라인 구현
             (개요(summary), 특징(Characteristic), 여담(digression) 으로 바꿔서 실행해봄)
- {{배정윤}}: 브랜치를 만들어서 독립적으로 실행하도록 빌드 해주면 됀다는 것! 여러가지의 기능들을 통합해서 서비스 제공할 경우, 병렬연결한 
             노드개수가 동일하고 워크 플로우가 동일하다면, 공통의 워크플로우 빌딩을 하고, 병렬 워킹할 노드들을 리스트로 만들어서 연결 가능하다! 다음에는 각 다른 기능들이 각자 다른 플로우인 경우로 해봐야겠근..

- 병렬화와 비동기가 서로 비슷하면서 다른 듯 한데 뭔가 차이점이 있나요??
  그리고 작업1: 3초 / 작업2: 2초 / 작업3: 5초 라고 한다면
  총 10초에 걸리는 작업을 병렬화하면 5초로 단축시킨다고 이해하면 되나요??
==> 아뇨 비동기와 병렬화는 다릅니다.
    비동기: 숙제 시켜두고, 일단 게임하는 것
    비동기: 작업을 하라고 해놓고, 기다리는 동안 다른 작업을 하는 것
    비동기가 많이 쓰이는 경우는: 응답이 오래 걸리는 이미지 생성의 경우 요청 보내놓고 다른 작업을 처리할 수 있는거고
    병렬화: 친구들 불러서 숙제 나눠서 하는 것
    병렬화: 한번에 여러 작업을 하는 것
    병렬화가 많이 쓰이는 경우는: 여러 문서를 요약할 때 쓸 수 있는거죠.

    오늘 예시에서 속도가 빨라졌다고 병렬화로 얻는 이점이라고 볼 수 있죠.
