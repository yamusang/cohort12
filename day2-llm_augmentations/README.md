## 자주 나온 질문
- git 올릴때 헷갈린다.
    - git fetch upstream\
      git merge upstream/main\
      git add .\
      git commit -m "커밋 내용"\
      git push origin main\
      내 레포에 코드가 다 올라갔으면 upstream쪽으로 pr요청

- 전역 .env와 안에 폴더 안에 들어가있는 .env의 역할과 스코프 차이
    - 큰 의미는 없는데요. 어제 폴더 안에 들어가 있던건 랭그래프 서버용으로 따로 준거라고 보시면 됩니다. 걔는 걔만의 서버가 생기는거라서요

- 구조화된 출력을 위한 증강을 했을 때 out 이 어디서 결정되서 나오는지
    - llm의 응답을 search_query 쿼리 형식에 맞게 나오도록 하는거라고 보시면 됩니다


- llm 증강이 무엇인지
    - 오늘 다룬 증강은 llm을 이용하는 확장 기능이라고 보시면 됩니다.
    - tool bind: llm이 해당 도구를 사용해서 말할 수 있는거죠

- LangChain에서 LLM 클래스는 모델 제공자별로 다르게 사용해야 한다는 점
    - init_chat_model 을 사용해도 됩니다. 제공자별로 제공하는 기능이 좀 다를수도 있어서요

- @tool 데코레이터의 역할
    - 데코레이터고 원래는 붙이는게 맞습니다. 이거 툴이야 이러면서요. 근데 이번 예제에서는 tool bind에만 집중했습니다. 헷갈린다 하시면 오늘 코드도 from langchain.tools import tool 하고 툴로 만든 함수명 위에 입력하시면 됩니다
